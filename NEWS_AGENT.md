# News Digest Agent

An autonomous Python agent that collects news from RSS feeds and news sources, summarizes them with the Claude API, and delivers a daily digest to Slack or email.

---

## Goal

Build a CLI script that runs automatically every morning (via GitHub Actions cron) and delivers a concise, readable news digest with summaries and source links.

---

## Tech Stack

| Component | Choice | Rationale |
|---|---|---|
| Runtime | Python 3.12 + uv | Fast setup, modern tooling |
| News | RSS via `feedparser` | Free, no API keys required |
| AI | Claude API (`claude-sonnet-4-5`) | Great for summarization, low cost |
| Delivery | Slack Incoming Webhook | Easiest to set up |
| Scheduling | GitHub Actions cron | Free, zero infrastructure |
| Configuration | `sources.yaml` + `.env` | Easy to add/remove sources |

## Phases

### Phase 1 — Fetch News from RSS

**Goal:** Script that fetches and prints articles from 3–5 RSS sources in the terminal.

**Tasks:**
- Initialize project with `uv init` and add dependencies: `feedparser`, `httpx`, `python-dotenv`, `pyyaml`
- Create `sources.yaml` with a few starting sources (see example below)
- Implement `fetcher.py`:
  - Read sources from `sources.yaml`
  - Fetch RSS feeds with `feedparser`
  - Return a list of article objects: `title`, `url`, `summary`, `published`, `category`
  - Filter out articles older than 24 hours
- Run and validate output in the terminal

**`sources.yaml` – example:**
```yaml
sources:
  - name: "SVT Nyheter"
    url: "https://www.svt.se/nyheter/rss.xml"
    category: "Sweden"
  - name: "TechCrunch"
    url: "https://techcrunch.com/feed/"
    category: "Tech"
  - name: "Hacker News"
    url: "https://hnrss.org/frontpage"
    category: "Tech"
  - name: "The Verge"
    url: "https://www.theverge.com/rss/index.xml"
    category: "Tech"
  - name: "Omni Ekonomi"
    url: "https://omni.se/rss"
    category: "Economy"
```

**Deliverable:** `python main.py` prints articles in the terminal.

### Phase 2 — Filter and Summarize with Claude

**Goal:** Agent that deduplicates articles and summarizes them with the Claude API per category.

**Tasks:**
- Add `anthropic` as a dependency
- Implement `filter.py`:
  - Deduplicate articles based on URL and title (fuzzy match on title with `difflib`)
  - Max number of articles per category (configurable, default 5)
- Implement `summarizer.py`:
  - Call `claude-sonnet-4-5` to summarize each article in 2–3 sentences in English
  - Group articles by category
  - Generate a short overall headline for the entire digest
  - Handle API errors with retry (max 3 attempts, exponential backoff)

**System prompt for Claude (starting point):**
```
You are a news editor writing daily briefings.
Summarize the article in 2–3 sentences in English.
Be factual, concise, and informative.
Avoid passive voice. No filler phrases.
If the article is not newsworthy, respond with: SKIP
```

**Deliverable:** `python main.py` prints a structured digest with summaries.

---

### Phase 3 — Delivery to Slack

**Goal:** Digest is delivered as a formatted Slack message.

**Tasks:**
- Create a Slack app with Incoming Webhook (slack.com/apps/manage)
- Add `SLACK_WEBHOOK_URL` to `.env`
- Implement `publisher.py`:
  - Build a Slack Block Kit message with sections per category
  - Each article: bold headline + summary + link
  - Add date and article count in the header
  - Send with `httpx.post()` to the webhook URL
- Handle HTTP errors and log clearly

**Slack Block Kit structure:**
```
[Header] 📰 News Digest – Tuesday June 3 · 12 articles

[Section] 🇸🇪 Sweden
  • Headline — Summary. Summary. [Read more →]
  • Headline — Summary. [Read more →]

[Divider]

[Section] 💻 Tech
  • ...

[Context] Generated by Claude · SVT, TechCrunch, HN
```

**Deliverable:** `python main.py` sends a real Slack message.

---

### Phase 4 — Scheduling with GitHub Actions

**Goal:** The agent runs automatically every weekday at 07:00.

**Tasks:**
- Create `.github/workflows/digest.yml`
- Configure cron schedule: `0 5 * * 1-5` (07:00 Swedish time = 05:00 UTC)
- Add GitHub Secrets: `ANTHROPIC_API_KEY`, `SLACK_WEBHOOK_URL`
- Set up Python environment in workflow with `uv`
- Add basic logging to `main.py` (timestamp, article count, delivery status)
- Test manually with `workflow_dispatch`

**`digest.yml` – basic structure:**
```yaml
name: Daily News Digest

on:
  schedule:
    - cron: '0 5 * * 1-5'
  workflow_dispatch:

jobs:
  digest:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: astral-sh/setup-uv@v3
      - run: uv sync
      - run: uv run python main.py
        env:
          ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
          SLACK_WEBHOOK_URL: ${{ secrets.SLACK_WEBHOOK_URL }}
```

**Deliverable:** Automatic digest every weekday without manual execution.

## Error Handling

- RSS errors: log and skip the source, continue with the rest
- Claude API errors: retry 3 times with backoff, fall back to the article's original `summary` from RSS
- Slack errors: log error with full response body
- No articles found: do not send message, log warning
